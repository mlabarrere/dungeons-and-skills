Export PDF — remplir une fiche de personnage

Methodologie pour remplir un PDF de fiche fourni a partir des donnees deja etablies (voir protocole). Chaque regle ci-dessous vient d'un bug reel rencontre : c'est un journal anti-regression. A appliquer integralement avant de livrer un PDF.

Principe : trois passes, jamais une seule

Un PDF n'est jamais « termine » apres une generation. Il subit trois controles independants sur la meme version finale :

  1. Donnees & regles : recalcul independant (modificateurs, JS, competences, CA, PV, DD/attaque de sort, PP), avant d'ecrire.
  2. Technique & visuel : rouvrir le fichier depuis le disque, rendre chaque page (200-300 dpi), zoomer les zones denses.
  3. Comparaison finale : relire les valeurs reellement presentes dans le PDF et les confronter a la table de reference.

Interdit : ecrire « c'est bon / verifie / parfait » sans les controles chiffres qui le prouvent.

Workflow de reference

  1. Analyse : lister les PDF, detecter AcroForm, inventorier chaque champ (nom, type, page, rectangle). Choisir le fichier et justifier. Enregistrer le sha256 de la source.
  2. Table de reference : recalculer toutes les valeurs (passe 1) avant d'ecrire. Aucune donnee inventee ; champ non defini = vide + liste des manques.
  3. Correspondance : pour chaque champ, etablir « libelle imprime → coordonnees → nom interne → valeur ». Se fier au libelle, pas au nom.
  4. Remplissage : une seule passe live ; AcroForm en priorite. Police sobre compatible FR (Helvetica), accents conserves. Tailles coherentes, auto-reduction sans passer sous un seuil lisible. Tirets pour les listes.
  5. Cases & listes verticales : cocher par position/libelle imprime, verifie par on_state().
  6. Sorts : annotation compacte et exacte quand ca tient (ex. Flamme sacree : JS Dex14, 1d8 rad), effet verifie sur la page du sort.
  7. Rendu : NeedAppearances=false, ne pas modifier la source, sortir un nom distinct, conserver pages et dimensions.
  8. Trois passes (ci-dessus) sur la version finale ; corriger et recommencer la passe concernee tant qu'il reste un ecart.

Checklist de livraison

Source inchangee (sha256) Pages & dimensions identiques Nom de fichier distinct et clair Chaque valeur en face du bon libelle imprime Cases cochees au bon endroit Accents FR presents et rendus Listes a tirets Sorts annotes 3 passes faites avec resultats chiffres Champs non definis laisses vides + listes
Une seule case non satisfaite interdit la livraison : corriger, puis recommencer les controles concernes.
